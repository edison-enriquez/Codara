# 3.1 Tipos de datos adicionales

---

## 3.1.1 Más allá del `int` básico

El tipo `int` que conocemos funciona bien para la mayoría de los casos, pero C ofrece variantes para situaciones específicas: cuando necesitas solo números positivos, cuando los números son muy grandes, o cuando necesitas ahorrar memoria.

---

## 3.1.2 El tipo `unsigned int`

Por defecto, un `int` puede ser positivo o negativo (tiene signo). El modificador `unsigned` le dice al compilador que la variable almacenará **solo valores no negativos**.

| Tipo | Rango aprox. (32 bits) |
|------|----------------------|
| `int` (con signo) | −2,147,483,648 a +2,147,483,647 |
| `unsigned int` | 0 a 4,294,967,295 |

```c exec
#include <stdio.h>

int main(void)
{
    unsigned int positivo = 4000000000u; // nota la 'u' al final
    int          normal   = 2000000;

    printf("unsigned int: %u\n", positivo); // usar %u, no %d
    printf("int normal:   %d\n", normal);

    return 0;
}
```

> **Advertencia:** Nunca asignes un valor negativo a un `unsigned`. El resultado es un número enorme (desbordamiento por debajo).

---

## 3.1.3 `short`, `long` y `long long`

C también permite modificar el **tamaño** del entero:

| Tipo | Tamaño típico | Rango aproximado |
|------|--------------|-----------------|
| `short int` | 2 bytes | −32,768 a +32,767 |
| `int` | 4 bytes | −2.1 billion a +2.1 billion |
| `long int` | 4 u 8 bytes | Igual o mayor que `int` |
| `long long int` | 8 bytes | −9.2×10¹⁸ a +9.2×10¹⁸ |

```c exec
#include <stdio.h>

int main(void)
{
    short     s = 30000;
    int       i = 2000000000;
    long long ll = 9000000000LL; // nota el sufijo LL

    printf("short:     %d\n",   s);
    printf("int:       %d\n",   i);
    printf("long long: %lld\n", ll); // %lld para long long

    return 0;
}
```

---

## 3.1.4 El modificador `signed`

Explícitamente declara que una variable tiene signo (es el comportamiento por defecto para `int` y `char`):

```c exec
#include <stdio.h>

int main(void)
{
    signed int   con_signo   = -100; // puede ser negativo
    unsigned int sin_signo   = 100;  // solo positivo

    signed char  byte_con    = -50;
    unsigned char byte_sin   = 200;

    printf("signed int:    %d\n", con_signo);
    printf("unsigned int:  %u\n", sin_signo);
    printf("signed char:   %d\n", byte_con);
    printf("unsigned char: %u\n", byte_sin);

    return 0;
}
```

---

## 3.1.5 El operador `sizeof`

Para conocer exactamente cuántos bytes ocupa un tipo en tu sistema, usa el operador `sizeof`:

```c exec
#include <stdio.h>

int main(void)
{
    printf("Tamaños en este sistema:\n");
    printf("  char:      %zu bytes\n", sizeof(char));
    printf("  short:     %zu bytes\n", sizeof(short));
    printf("  int:       %zu bytes\n", sizeof(int));
    printf("  long:      %zu bytes\n", sizeof(long));
    printf("  long long: %zu bytes\n", sizeof(long long));
    printf("  float:     %zu bytes\n", sizeof(float));
    printf("  double:    %zu bytes\n", sizeof(double));

    return 0;
}
```

> **Nota:** `sizeof` devuelve un tipo `size_t`, por eso se usa `%zu` para imprimirlo.

---

## 3.1.6 ¿Cuándo usar cada tipo?

| Situación | Tipo recomendado |
|-----------|-----------------|
| Contadores simples, índices | `int` |
| Edades, cantidades siempre positivas | `unsigned int` |
| IDs de bases de datos grandes | `long long` |
| Códigos de color, bytes de datos | `unsigned char` |
| Índices en arreglos pequenos | `short` |
| Cálculos matemáticos de precisión | `double` |

---

## Resumen

| Tipo | Descripción |
|------|-------------|
| `unsigned int` | Solo valores ≥ 0; doble rango positivo |
| `short int` | Entero pequeño (2 bytes) |
| `long int` | Entero grande (al menos 4 bytes) |
| `long long int` | Entero muy grande (8 bytes) |
| `signed` | Con signo (por defecto en `int` y `char`) |
| `sizeof(tipo)` | Devuelve el tamaño en bytes |
| `%u` | Formato para `unsigned int` |
| `%lld` | Formato para `long long` |
| `%zu` | Formato para `size_t` |
