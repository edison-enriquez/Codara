# 1.1 El viaje comienza: Tu primer programa

## El programa más simple

Queremos que un texto corto aparezca en la pantalla:

```
It's me, your first program.
```

El **algoritmo** (descripción estructurada de pasos) sería:

1. **iniciar**
2. **escribir** el texto en la pantalla
3. **detenerse**

Y así se ve el programa real en C:

```c exec
#include <stdio.h>

int main(void)
{
    puts("It's me, your first program.");
    return 0;
}
```

## Línea por línea

### La directiva `#include`

```c
#include <stdio.h>
```

El `#` indica una **directiva del preprocesador**. Esta directiva le dice al compilador que incluya el contenido del archivo `stdio.h` (Standard Input/Output).

`stdio.h` contiene declaraciones de funciones para escribir en pantalla y leer del teclado. Sin ella, el compilador no sabría qué es `puts()`.

### La función `main`

```c
int main(void)
{
    // código aquí
}
```

El estándar de C requiere que siempre exista una función llamada `main`. Es el **punto de entrada** del programa.

- `int` → el resultado de la función es un número entero
- `main` → el nombre (obligatorio)
- `void` → no recibe parámetros

### La instrucción `puts`

```c
puts("It's me, your first program.");
```

`puts` es una **función** que escribe una cadena de texto en pantalla (añade un salto de línea al final automáticamente).

Los textos entre comillas `"..."` se llaman **cadenas de caracteres** o *strings*.

### El `return 0`

```c
return 0;
```

`return` termina la función y devuelve un valor. El `0` le indica al sistema operativo que el programa terminó **correctamente** (sin errores).

## Los bloques (llaves `{}`)

Las llaves `{` y `}` delimitan el **cuerpo** de una función. Todo lo que está entre ellas pertenece a esa función.

```c
int main(void)
{                           // ← inicio del cuerpo
    puts("Hola mundo");
    return 0;
}                           // ← fin del cuerpo
```

## Intentemos variaciones

Puedes llamar a `puts` varias veces:

```c exec
#include <stdio.h>

int main(void)
{
    puts("Mi primer programa.");
    puts("Segunda línea.");
    puts("Tercera línea.");
    return 0;
}
```

> 💡 **Tip:** Edita el texto entre comillas y vuelve a ejecutar para ver el cambio.

## Errores comunes

| Error | Causa |
|-------|-------|
| Olvidar el `;` al final | C requiere `;` al final de cada instrucción |
| No incluir `stdio.h` | El compilador no reconoce `puts` ni `printf` |
| Usar `MAIN` en vez de `main` | C es sensible a mayúsculas (*case-sensitive*) |
| Olvidar las llaves `{}` | El cuerpo de la función debe estar delimitado |

---

> **¿Sabías que?** El compilador GCC puede darte mensajes de error muy detallados. Usa la opción `-Wall` para ver todas las advertencias: `gcc -Wall programa.c -o programa`
