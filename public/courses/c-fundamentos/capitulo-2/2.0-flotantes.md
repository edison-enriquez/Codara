# 2.0 Los números de punto flotante

---

## 2.0.1 ¿Qué son los números de punto flotante?

En la lección anterior conocimos el tipo `int` para números enteros. Ahora es momento de hablar de otro tipo diseñado para representar **números que tienen una parte fraccionaria**.

En otras palabras, son los números que tienen (o pueden tener) una parte decimal después del punto. Cuando usamos expresiones como "dos y medio" o "cero punto cuatro", estamos pensando en lo que una computadora llama **números de punto flotante** (*floating-point numbers*).

---

## 2.0.2 Cómo escribir números de punto flotante en C

Si quieres usar el valor "dos y medio", debes escribirlo así:

```
2.5
```

> **Regla importante:** el punto decimal es **obligatorio** para que el compilador reconozca un número como floating-point. No se usa coma, sino punto.

Puedes omitir el cero cuando es el único dígito antes o después del punto:

```c
0.4  →  .4   // equivalentes
4.0  →  4.   // equivalentes
```

### Entero vs flotante: diferencia crucial

```c exec
#include <stdio.h>

int main(void)
{
    int i;
    float x;

    i = 10 / 4;     // división ENTERA: resultado = 2
    x = 10.0 / 4.0; // división FLOTANTE: resultado = 2.5

    printf("Entero:  %d\n", i);
    printf("Flotante: %f\n", x);
    return 0;
}
```

**Un número que tiene punto es `double`. Un número sin punto es `int`. El punto hace la diferencia.**

---

## 2.0.3 Notación científica

Para números muy grandes o muy pequeños, C permite la notación científica con la letra `e` (exponente):

| Forma matemática | En C |
|-----------------|------|
| 3 × 10⁸ | `3e8` |
| 3.0 × 10⁸ | `3.0E8` |
| 6.626 × 10⁻³⁴ | `6.626e-34` |

```c exec
#include <stdio.h>

int main(void)
{
    double luz = 3e8;          // velocidad de la luz (m/s)
    double planck = 6.626e-34; // constante de Planck

    printf("Velocidad de la luz: %e m/s\n", luz);
    printf("Constante de Planck: %e J·s\n", planck);
    return 0;
}
```

La letra `e` es una abreviatura de "multiplicado por diez a la potencia de".

---

## 2.0.4 Tipos `float` y `double`

C tiene dos tipos principales para números de punto flotante:

| Tipo | Precisión | Tamaño | Uso recomendado |
|------|-----------|--------|-----------------|
| `float` | ~6 dígitos decimales | 4 bytes | Cálculos simples donde el espacio importa |
| `double` | ~15 dígitos decimales | 8 bytes | **Recomendado** para la mayoría de casos |

```c exec
#include <stdio.h>

int main(void)
{
    float  PI_f = 3.14159;       // float: menos preciso
    double PI_d = 3.14159265358979; // double: más preciso

    printf("float:  %.5f\n", PI_f);
    printf("double: %.14f\n", PI_d);
    return 0;
}
```

> **Regla práctica:** Usa `double` por defecto. Solo usa `float` si necesitas ahorrar memoria en sistemas embebidos o arreglos muy grandes.

---

## 2.0.5 Conversión entre `int` y `float`

La transformación de `int` a `float` es siempre posible, pero puede causar **pérdida de precisión**.

```c exec
#include <stdio.h>

int main(void)
{
    int   i = 100;
    float f = 100.25;

    // int → float: siempre funciona
    f = i;        // f ahora es 100.0 (conversión automática)
    printf("int a float: %f\n", f);

    // float → int: se PIERDE la parte decimal
    i = 100.25;   // i ahora es 100 (se trunca el .25)
    printf("float a int: %d\n", i);

    return 0;
}
```

> ⚠️ Al convertir `float` a `int`, la parte decimal se **trunca** (no se redondea). El `.25` simplemente desaparece.

Además, los enteros tienen un tamaño limitado. Si intentas asignar un número flotante muy grande a un `int`, el resultado es **impredecible** (depende del sistema).

## Resumen

| Concepto | Detalle |
|----------|---------|
| `float` | Punto flotante de precisión simple (6 dígitos) |
| `double` | Punto flotante de precisión doble (15 dígitos) — preferido |
| El punto decimal | Distingue `4` (int) de `4.0` (double) |
| `%f` | Especificador de formato para float/double |
| `%e` | Especificador de formato en notación científica |
| Notación `e` | `3e8` = 3 × 10⁸ |
