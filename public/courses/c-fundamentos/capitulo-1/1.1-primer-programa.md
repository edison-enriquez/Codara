# 1.1 El viaje comienza: Creando tu primer programa

---

## 1.1.1 Tu primer programa

Ahora nos gustaría mostrarte un programa muy simple escrito en el lenguaje "C". Queremos que el siguiente texto aparezca en la pantalla:

```
It's me, your first program.
```

¿Qué pasos debería realizar nuestro primer programa? Intentemos enumerarlos:

1. **iniciar**
2. **escribir** el texto en la pantalla
3. **detenerse**

Este tipo de descripción estructurada y semi-formal de cada paso del programa se llama un **algoritmo**. Las fuentes de esta palabra se pueden rastrear hasta el idioma árabe, y se originó a principios de la época medieval.

Ahora veamos el programa en el editor:

```c exec
#include <stdio.h>

int main(void)
{
    puts("It's me, your first program.");
    return 0;
}
```

Se ve un poco misterioso, ¿no? Ahora miraremos cuidadosamente cada línea del programa, explicando su significado y propósito.

---

## 1.1.2 La directiva `#include`

Presta atención al carácter `#` al inicio de la primera línea. Significa que el contenido de esta línea es una **directiva del preprocesador**.

El **preprocesador** es una parte separada del compilador cuya tarea es pre-leer el texto del programa y hacer algunas modificaciones **antes** de que comience la compilación real.

La directiva `include` le dice al preprocesador que reemplace esa línea con el contenido del archivo `stdio.h`. Los cambios del preprocesador nunca modifican tu archivo fuente original.

¿Por qué necesitamos `stdio.h`? Escribir un programa es similar a construir un edificio con bloques prefabricados. Vamos a usar un bloque llamado `puts` para escribir en la pantalla, pero el compilador no sabe nada de él todavía. El archivo **stdio.h** (*Standard Input/Output*) contiene la información preliminar que el compilador necesita conocer sobre `puts` y muchas otras funciones de entrada/salida.

---

## 1.1.3 La función `main`

Ya hemos mencionado algo sobre **bloques**. Uno de los tipos más comunes de bloques en "C" son las **funciones**. Imagina una función como una **caja negra**: puedes insertar algo (argumentos) y obtener un resultado (valor de retorno).

El estándar del lenguaje "C" requiere que siempre exista una función con el nombre `main`; sin ella el programa será incorrecto. Esta función es el **punto de entrada** del programa.

Cada función en "C" comienza con la siguiente información:

- ¿cuál es el **resultado** de la función?
- ¿cuál es el **nombre** de la función?
- ¿cuántos **parámetros** tiene?

```c
int main(void)
{
    puts("It's me, your first program.");
    return 0;
}
```

- `int` → el resultado es un **valor entero** (*integer*)
- `main` → el nombre (obligatorio)
- `void` → la función **no requiere parámetros**

A este conjunto de información se le llama **prototipo**. El interior de la función se llama **cuerpo** y está delimitado por llaves `{` y `}`. Incluso puede estar vacío; por ejemplo, una función perezosa:

```c
void lazy(void) { }
```

---

## 1.1.4 La instrucción `puts`

Dentro del cuerpo de `main` encontramos una **invocación de función**: `puts`.

```c
puts("It's me, your first program.");
```

Puntos importantes:

- Cada sentencia en "C" **debe terminar con punto y coma** `;`. Sin él, el programa será incorrecto.
- `puts` es una abreviatura de "*PUT String*" (poner cadena). Muestra el texto en pantalla y añade un salto de línea automáticamente.
- El nombre de la función invocada siempre debe ir seguido de paréntesis `()`, incluso cuando no espera parámetros.
- Las cadenas de texto entre comillas `"..."` se llaman **cadenas de caracteres** (*strings*). Las comillas no se muestran en pantalla; le indican al compilador que es texto.

---

## 1.1.5 El `return 0`

```c
return 0;
```

Esta sentencia **termina la ejecución de la función**. El `0` es el resultado de `main` y le comunica al sistema operativo:

> *"Hice lo que tenía que hacer, nada me detuvo y todo está bien."*

Si escribieras `return 1;` significaría que algo salió mal. El sistema operativo puede usar esta información para reaccionar de la manera más apropiada.

---

## Resumen: el programa completo

```c exec
#include <stdio.h>

int main(void)
{
    puts("It's me, your first program.");
    return 0;
}
```

Paso a paso:

1. `#include <stdio.h>` — incluye las declaraciones necesarias para `puts`.
2. `int main(void)` — define la función principal (punto de entrada).
3. `puts(...)` — imprime el texto en la pantalla.
4. `return 0;` — termina el programa indicando que todo salió bien.

Prueba llamar a `puts` varias veces:

```c exec
#include <stdio.h>

int main(void)
{
    puts("Primera línea.");
    puts("Segunda línea.");
    puts("Tercera línea.");
    return 0;
}
```
