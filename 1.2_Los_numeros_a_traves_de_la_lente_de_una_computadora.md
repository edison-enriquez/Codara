# 1.2 Los números a través de la lente de una computadora

---

## 1.2.1 Los números y cómo los ven las computadoras

¿Sabes cómo las computadoras realizan cálculos con números? Quizás hayas oído hablar del **sistema binario** y sepas que es el sistema que las computadoras usan para almacenar números, y que pueden realizar cualquier operación sobre ellos. No vamos a explorar las complejidades de los sistemas numéricos posicionales aquí, pero diremos que los números manejados por las computadoras modernas son de dos tipos:

- **enteros** (*integers*), es decir, aquellos que carecen de parte fraccionaria;
- **números de punto flotante** (*floating-point*), o simplemente flotantes (*floats*), que contienen (o son capaces de contener) la parte fraccionaria.

Esta definición no es completamente precisa pero es suficiente para nuestros propósitos. Esta distinción es muy importante y el límite entre estos dos tipos de números es **muy estricto**. Ambos tipos de números difieren significativamente en cómo se **almacenan** en la memoria de una computadora y en el rango de valores aceptables. Además, la característica de un número que determina su tipo, rango y aplicación se llama **tipo**.

En este punto hemos hecho amistad con dos tipos del lenguaje "C": un **tipo entero** (conocido como ***int***) y un **tipo de punto flotante** (conocido como ***float***).

Por ahora, dejemos los números de punto flotante a un lado (volveremos a ellos pronto) y consideremos la pregunta, quizás un poco banal a primera vista, de cómo el lenguaje "C" reconoce los enteros.

Bueno, es casi de la misma manera en que estás acostumbrado a escribirlos con un lápiz en papel: es simplemente una **cadena de dígitos** que componen el número. Pero hay una reserva: no debes insertar ningún carácter que no sea dígito dentro del número. Tomemos por ejemplo el número once millones ciento once mil ciento once. Si tomaras un lápiz en tu mano ahora mismo, escribirías el número así:

```
11,111,111
```

Obviamente, esto facilita la lectura si el número está compuesto por muchos dígitos. Sin embargo, en "C" está prohibido. **Debes** escribir este número de la siguiente manera:

```
11111111
```

De lo contrario, te expondrías a algunos comentarios mordaces del compilador. ¿Cómo codificamos números negativos en "C"? Como de costumbre: añadiendo un signo menos. Puedes escribir:

```
-11111111
```

Los números positivos no necesitan estar precedidos por el signo más, pero puedes hacerlo si quieres. Las siguientes líneas describen el mismo número:

```
+123
123
```

Por ahora, trataremos solo con enteros; introduciremos los números de punto flotante en el próximo capítulo.

Hay dos convenciones adicionales, desconocidas en el mundo de las matemáticas. La primera nos permite usar los números en una representación octal. Si un número entero está precedido por el dígito *0*, será tratado como un **valor octal**. Esto significa que el número solo debe contener dígitos del rango *[0..7]*.

```
0123
```

es un número octal con un valor decimal igual a 83.

```
0x123
```

es un **número hexadecimal** con un valor decimal igual a 291.

Quizás quieras ver el resultado de tu cómputo. Discutiremos esto más adelante, pero ahora es un buen momento para mencionar cómo imprimir el valor de un número.

Para imprimir un número entero, deberías usar (esta es solo una forma simple):

```c
printf("%d\n", NumeroEnteroOExpresion);
```

Para imprimir un número de punto flotante, deberías usar (esta es solo una forma simple):

```c
printf("%f\n", NumeroFlotanteOExpresion);
```

En ambos casos, primero debes incluir el archivo de encabezado stdio (como hicimos en el primer programa):

```c
#include <stdio.h>
```

## 1.2.2 Una variable es variable

Parece bastante obvio que el lenguaje "C" nos permite escribir números. No te sorprenderá que podamos hacer algunas operaciones aritméticas con estos números: sumarlos, restarlos, multiplicarlos y dividirlos. Lo haremos pronto. Pero es perfectamente normal preguntarse cómo almacenar los resultados de estas operaciones para usarlos en otras operaciones. Existen "**contenedores**" especiales para este propósito y estos contenedores se llaman **variables**. El nombre sugiere que el contenido de un contenedor puede ser variado de (casi) cualquier manera.

¿Qué tiene cada variable?

- un **nombre**;
- un **tipo**;
- un **valor**;

---

## 1.2.3 Nombres de variables

Comencemos nuestra discusión con los temas relacionados con el **nombre** de una variable. Las variables no aparecen en nuestro programa de forma mágica. Nosotros (como desarrolladores) decidimos cuántas y cuáles variables queremos que existan en nuestro programa. También les damos sus nombres, casi convirtiéndonos en sus padrinos. Si quieres dar un nombre a una variable debes seguir algunas reglas estrictas:

- el nombre de la variable debe estar compuesto de **letras latinas mayúsculas o minúsculas, dígitos y el carácter** _ (*guion bajo* o *underscore*);
- el nombre de la variable debe **comenzar con una letra**;
- el **carácter de guion bajo es una letra** (extraño pero cierto);
- las letras mayúsculas y minúsculas se tratan como **diferentes** (un poco diferente al mundo real: Alice y ALICE son los mismos nombres propios, pero son dos nombres de variables diferentes, y por lo tanto, dos variables diferentes);

Ten en cuenta que las mismas restricciones se aplican a los **nombres de funciones**.

El estándar del lenguaje "C" no impone restricciones sobre la longitud de los nombres de variables, pero un compilador específico puede tener una opinión diferente al respecto.

Algunos nombres de variables **correctos**: variable, i, t10, Exchange_Rate, counter, DaysToTheEndOfTheWorld, _

Algunos nombres **incorrectos**: 10t (no comienza con una letra), Adiós_Señora (contiene caracteres ilegales), Exchange Rate (contiene un espacio).

---

## 1.2.4 Tipos de variables

El *tipo* es un **atributo** que define de manera única qué valores pueden almacenarse dentro de la variable. Ya conocemos los tipos entero (*int*) y de punto flotante (*float*). El valor de una variable es lo que hemos puesto en ella. Por supuesto, solo puedes poner un valor que sea compatible con el tipo de la variable. Solo un valor entero puede asignarse a una variable entera. El compilador no nos permitirá poner un número de punto flotante allí.

Hablemos ahora de dos cosas importantes: cómo se crean las variables y cómo poner un valor dentro de ellas.

La variable cobra existencia como resultado de una **declaración**. Una declaración es una estructura sintáctica que vincula un nombre, proporcionado por el programador, a un tipo específico ofrecido por el lenguaje "C". La construcción de la declaración es simple: solo usa el nombre del tipo deseado, luego el nombre de la variable (o nombres de variables separados por comas si hay más de una). Toda la sentencia termina con un punto y coma.

---

## 1.2.5 Declarando variables

Intentemos declarar una variable de tipo *int* llamada *Counter*:

```c
int Counter;
```

¿Qué declara el siguiente fragmento de programa?

```c
int variable1, account_balance, invoices;
```

**Respuesta:** Declara tres variables de tipo *int* llamadas (respectivamente) variable1, account_balance e invoices.

Recuerda que puedes usar tantas declaraciones de variables como necesites para lograr tu objetivo.

---

## 1.2.6 Valores de variables

¿Y cómo le damos un valor a la variable recién declarada? Necesitas usar el **operador de asignación**: `=`

Ejemplos:

```c
Counter = 1;
```

Lee: asignar un valor de `1` a una variable llamada `Counter`.

```c
result = 100 + 200;
```

El nuevo valor de `result` será el resultado de sumar `100` a `200`.

```c
x = x + 1;
```

Un matemático protestaría: ningún valor puede ser igual a sí mismo más uno. Pero en "C" el signo `=` no significa *es igual a*, sino **asignar un valor**. Toma el valor actual de `x`, súmale `1` y almacena el resultado en `x`. El valor de `x` fue **incrementado** en uno.

---

## 1.2.7 Palabras clave – por qué son las claves

Se llaman **palabras clave** (*keywords*), o más precisamente, **palabras reservadas** (*reserved keywords*).

Están reservadas porque no debes usarlas como nombres: ni para tus variables, ni funciones, ni ninguna otra entidad nombrada.

Como el compilador de "C" es **sensible a mayúsculas y minúsculas** (*case-sensitive*), puedes modificar cualquiera de estas palabras cambiando el caso de cualquier letra, creando así una nueva palabra que ya no está reservada.

Por ejemplo, no puedes hacer esto: `int int;` ❌

Pero puedes hacer esto: `int Int;` ✅

## 1.2.8 Comentarios en C

Los comentarios son notas dirigidas no al compilador sino a los humanos, para **explicar** cómo funciona el código, los significados de variables y funciones, y para conservar información sobre quién es el autor y cuándo se escribió el programa.

Cada comentario es léxicamente equivalente a **un espacio**.

En "C" un comentario comienza con `/*` y termina con `*/`. Puede abarcar varias líneas o solo una.

```c
/* La variable Counter cuenta el número de ovejas en el prado */
int Counter;
```

```c
/*************************************************
Contando ovejas versión 1.1
Autor: Ronald Sleepyhead, 2024
email: rs@insomnia.org

Cambios:
2024-03-01: Ginny Drowsy: Conteo de ovejas negras mejorado
*************************************************/
```

Los comentarios **anidados** (`/* ... /* ... */ ... */`) pueden causar problemas según el compilador, por lo que se recomienda evitarlos.
